# 📂 Техническое SEO

**Эта папка — для ответственного за техническую часть и SEO.**

Что здесь: по каким запросам бить, насколько конкурентна выдача, как технически устроить мультиязычный сайт.

| Файл | Содержимое |
|---|---|
| `04b_seo_keywords.md` | Таблица ключевых запросов (интент/стадия/спрос-оценка/конкуренция), банк PAA-вопросов для FAQ, SERP-анализ (кто ранжируется), требования E-E-A-T/YMYL |
| `09_multilingual_architecture.md` | hreflang, структура локалей EN/UK/RU (субдиректории), canonical, sitemap, Search Console/Яндекс, дубли vs локализация, фазный план запуска, чек-лист частых ошибок |
| `10_seo_keywords_spanish.md` | Испанские запросы, PAA, региональные различия (ES/MX/AR), US-Hispanic + аудиокниги, каналы дистрибуции, E-E-A-T |

**Главное (не перепутать):**
- Код украинского — **`uk`, не `ua`**. Русский — `ru`, английский — `en`.
- Субдиректории `/en/ /uk/ /ru/`, корень `/` = x-default; hreflang self-referencing + reciprocal + x-default.
- **Не запускать локаль, пока контент не переведён и локализован целиком** — тонкие локали тянут вниз весь сайт (Helpful Content — site-wide).
- Тема **YMYL** → E-E-A-T (lived experience автора + ревью специалиста + источники) — технический приоритет.
- Объёмы запросов — **оценки** (без Ahrefs/SEMrush); перед инвестициями валидировать платными инструментами.
- Контент-план под эти запросы — в соседней папке `../02_content/04a_content_strategy.md`.

---

**📖 Термины:** **hreflang** — разметка языковых версий страницы; **локаль** — языко-региональная версия (`/en/ /uk/ /ru/`; код украинского — `uk`, не `ua`); **E-E-A-T** — опыт/экспертиза/авторитетность/доверие (критерии Google); **YMYL** — темы «здоровье/деньги» с макс. требованиями; **PAA** — блок «Похожие вопросы»; **intent (интент)** — намерение за запросом; **long-tail / head** — узкий / широкий запрос; **DA** — авторитет домена. (Полные глоссарии — в `04b` и `09`.)
